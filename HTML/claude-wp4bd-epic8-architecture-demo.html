// Epic 7 V2-060: WordPress Post Object Bridge
function wp4bd_node_to_post($node) {
  if (!is_object($node) || !isset($node->nid)) {
    return NULL;
  }

  $post = new WP_Post();

  // Map all 21 WP_Post properties
  $post->ID = (int) $node->nid;
  $post->post_author = isset($node->uid) ? (int) $node->uid : 0;
  $post->post_title = isset($node->title) ? $node->title : '';
  $post->post_type = isset($node->type) ? $node->type : 'post';
  $post->post_status = ($node->status == 1) ? 'publish' : 'draft';

  // Extract body content from Backdrop's field structure
  $post->post_content = '';
  if (isset($node->body[LANGUAGE_NONE][0]['value'])) {
    $post->post_content = $node->body[LANGUAGE_NONE][0]['value'];
  }

  // Extract excerpt/summary
  $post->post_excerpt = '';
  if (isset($node->body[LANGUAGE_NONE][0]['summary'])) {
    $post->post_excerpt = $node->body[LANGUAGE_NONE][0]['summary'];
  }

  // ... map remaining properties ...

  return $post;
}

ğŸ”‘ Key Technical Achievements

1. Complete Database Isolation

WordPress cannot connect to a database. All queries are intercepted by the db.php drop-in and return empty results. No SQL reaches MySQL.

// db.php drop-in
class wpdb extends wpdb_original {
  public function query($sql) {
    // Intercept! Don't execute.
    return false;
  }

  public function get_results($sql) {
    // Intercept! Return empty.
    return array();
  }
}

2. Real WordPress Objects

We use WordPress's actual classes from WordPress 4.9 core, not stubs or mocks:

  • WP_Post - From wp-includes/class-wp-post.php
  • WP_User - From wp-includes/class-wp-user.php
  • WP_Term - From wp-includes/class-wp-term.php
  • WP_Query - From wp-includes/class-wp-query.php

WordPress themes see genuine WordPress objects, not Backdrop objects dressed up.

3. Zero Conflicts

WordPress and Backdrop coexist without namespace collisions:

  • WordPress loads after Backdrop bootstrap (FULL phase)
  • WordPress constants scoped to prevent conflicts
  • No function name collisions between systems
  • Both systems' hooks work independently

4. Production Ready

The production template (page.tpl.php) includes:

  • Error handling with helpful troubleshooting messages
  • Debug mode accessible via ?wp4bd_debug=2
  • Graceful fallbacks for missing data
  • Integration of Backdrop assets ($styles, $scripts, $messages)
  • Calls to wp_head() and wp_footer() for theme compatibility
  • Clean, documented, maintainable code
